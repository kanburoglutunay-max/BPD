# Frontend Style Guide

## UX Direction

The interface should feel:

- premium
- institutional
- organized
- calm
- technical
- trustworthy

It should not feel playful, crowded, or experimental.

## Layout Principles

- prefer modular layouts
- avoid single-page overload
- keep BPMN and planning views connected but visually separated
- use side panels, tabs, and clear sections

## Recommended Screen Structure

### App Shell
- left sidebar
- top bar
- main content area
- right contextual panel when needed

### Dashboard
- overview cards
- recent processes
- draft BPMN candidates
- recent meetings or notes later

### Process Detail
- overview tab
- intake tab
- BPMN tab
- planning tab

### BPMN Review Screen
- center canvas
- left structure/process navigation
- right properties panel

## Sidebar Rules

- dark background
- minimal iconography
- active item with subtle gold emphasis
- no excessive decoration

## Topbar Rules

- keep it clean
- show context
- show primary actions
- avoid clutter

## Form Rules

- use grouped sections
- do not create long chaotic forms
- separate process info, roles, tasks, and decisions clearly
- use helper text only where necessary

## BPMN UX Rules

- generated BPMN must be editable
- selection must be visually clear
- node metadata should be easy to inspect
- process logic should be understandable even before manual editing

## Data Display Rules

- use clean cards and tables
- avoid too many colors
- use status colors carefully
- maintain hierarchy and spacing

## Do Not Do

- do not cram intake, BPMN, analytics, and comments onto one screen
- do not use bright unrelated colors
- do not overuse gold
- do not create generic template-looking SaaS UI
- do not let the interface look like a hackathon demo
